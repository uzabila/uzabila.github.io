Запросы к базе данных при смене домена WordPress-сайта

При переезде сайта на WordPress или смене домена иногда нужно внести изменения в базу данных сайта, чтобы все посты и сайт в целом нормально работал.

Для этого нам необходимо сделать три запроса к базе данных. Можно это сделать через phpmyadmin — это самый простой способ.

UPDATE wp_options SET option_value = replace(option_value, 'http://domain.ru', 'http://newdomain.ru') WHERE option_name = 'home' OR option_name = 'siteurl';

UPDATE wp_posts SET guid = replace(guid, 'http://domain.ru','http://newdomain.ru');

UPDATE wp_posts SET post_content = replace(post_content, 'http://domain.ru', 'http://newdomain.ru');

Меняем domain.ru и newdomain.ru на свои и всё.

В общем, если вы делаете перенос сайта на WordPress на другой домен, то действия выше обязательны, иначе будет ошибка обращения к Базе Данных.

Этот пост, на самом деле, из разряда постов для себя, т.к. довольно часто свои проекты или клиентские приходится переносить на новые домены.
